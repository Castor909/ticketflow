# TicketFlow — Manual Testing Checklist

## Подготовка

- [ ] Запустить бэкенд: `cd backend && npm run dev` → должен ответить `TicketFlow API running on http://localhost:3001`
- [ ] Запустить фронтенд: `cd frontend && npm run dev` → открыть http://localhost:5173
- [ ] Убедиться, что страница загружается без ошибок в консоли браузера

---

## 1. Публичный доступ (без авторизации)

- [ ] Открыть главную страницу — виден каталог событий (или пустой список, если событий нет)
- [ ] Navbar показывает: `Events`, `Log in`, `Sign up`
- [ ] Перейти на `/my-tickets` вручную → должен перекинуть на `/login`

---

## 2. Регистрация

### Организатор
- [ ] Нажать **Sign up**
- [ ] Заполнить: имя, email, пароль (мин. 6 символов), выбрать **Organiser**
- [ ] Нажать **Create account** → редирект на `/organiser`
- [ ] Navbar показывает имя пользователя и кнопку **Log out**

### Ошибки регистрации
- [ ] Попробовать зарегистрировать тот же email повторно → ошибка `Email already registered`
- [ ] Попробовать пароль из 3 символов → ошибка `Password must be at least 6 characters`

---

## 3. Логин / Логаут

- [ ] Нажать **Log out** → navbar возвращается в гостевой вид
- [ ] Нажать **Log in**, ввести правильные данные → успешный вход
- [ ] Ввести неправильный пароль → ошибка `Invalid credentials`
- [ ] Логаут и снова логин

---

## 4. Создание события (Organiser)

- [ ] Войти как организатор → открыть `/organiser`
- [ ] Нажать **+ New Event**
- [ ] Заполнить форму:
  - Title: `Summer Jazz Festival`
  - Description: любое описание
  - Category: `Music`
  - Location: `Palma de Mallorca`
  - Date: любая будущая дата
  - Price: `15.00`
  - Total tickets: `50`
- [ ] Нажать **Create event** → редирект на dashboard
- [ ] Событие видно в таблице: `0 / 50` продано
- [ ] Создать второе событие: бесплатное (`Price: 0`), категория `Conference`, tickets: `10`

---

## 5. Каталог событий

- [ ] Открыть главную — оба события видны в виде карточек
- [ ] Карточки показывают: название, категорию, дату, локацию, цену, количество оставшихся билетов
- [ ] **Поиск:** ввести `Jazz` → показывает только первое событие
- [ ] **Фильтр по категории:** выбрать `Conference` → показывает только второе
- [ ] **Фильтр по локации:** ввести `Palma` → показывает первое
- [ ] Нажать **Clear** → все фильтры сброшены, оба события видны

---

## 6. Страница события

- [ ] Кликнуть на карточку первого события → открывается `/events/:id`
- [ ] Видны: название, категория, дата, время, локация, организатор, описание
- [ ] Справа: цена `€15.00`, количество оставшихся, выпадающий список количества, кнопка **Buy tickets**
- [ ] Нажать **Buy tickets** без авторизации → редирект на `/login`

---

## 7. Покупка билетов (Attendee)

- [ ] Открыть новый браузер/вкладку в инкогнито
- [ ] Зарегистрироваться как **Attendee** (другой email)
- [ ] Вернуться на страницу `Summer Jazz Festival`
- [ ] Выбрать количество: **2**, нажать **Buy tickets**
- [ ] Появляется блок `Purchase confirmed!` с **2 уникальными кодами** (12 символов, формат `A1B2C3D4E5F6`)
- [ ] Ссылка **View all my tickets →** ведёт на `/my-tickets`

---

## 8. История покупок

- [ ] Открыть `/my-tickets` (или через navbar)
- [ ] Видна карточка покупки: название события, дата, локация
- [ ] Внизу карточки — коды купленных билетов
- [ ] Купить ещё раз другое (бесплатное) событие → в `/my-tickets` теперь 2 записи

---

## 9. Статистика организатора

- [ ] Войти снова как организатор
- [ ] Открыть `/organiser`
- [ ] Карточки статистики:
  - `Total events: 2`
  - `Tickets sold: 2`
  - `Revenue: €30.00`
- [ ] Таблица показывает для первого события: `2 / 50`

---

## 10. Редактирование и удаление событий

- [ ] Нажать **Edit** у первого события
- [ ] Изменить название → **Save changes** → изменение отражается в dashboard и каталоге
- [ ] Нажать **Delete** у второго события → подтвердить → событие исчезает из списка

---

## 11. Проверка защиты роутов

- [ ] Войти как **attendee**, попробовать открыть `/organiser` → редирект на `/`
- [ ] Без авторизации открыть `/my-tickets` → редирект на `/login`

---

## 12. Граничные случаи

- [ ] Создать событие с **1 билетом**, купить его → карточка показывает **SOLD OUT**
- [ ] Попробовать снова купить билет на тот же sold-out ивент → ошибка `Not enough tickets available`
- [ ] Попробовать купить больше билетов, чем доступно (если осталось 3, выбрать 5) → ошибка
